# 文档国际化规范

MatterLoop 的公开 Markdown 同时维护简体中文和英文。国际化只作用于面向用户的文档，不改变 Python
标识符、配置字段、事件值或源码注释语言。

## 语言与文件约定

简体中文是原路径，英文使用同目录 `.en.md` 镜像：

| 中文 | 英文 |
| --- | --- |
| `README.md` | `README.en.md` |
| `CHANGELOG.md` | `CHANGELOG.en.md` |
| `docs/architecture.md` | `docs/architecture.en.md` |
| `matterloop-core/README.md` | `matterloop-core/README.en.md` |

保留中文原路径可以避免已有 GitHub 链接、PyPI `project.readme` 和外部引用失效。新增公开 Markdown
必须在同一个提交中增加英文镜像。

## 页面语言入口

每个中文页面第一行使用：

```markdown
简体中文 | [English](对应英文文件)
```

英文页面第一行使用：

```markdown
[简体中文](对应中文文件) | English
```

仓库根文档、`docs/` 和示例可以使用相对链接。发行包 README 会作为 PyPI 项目说明渲染，因此语言
入口使用 `https://github.com/huleidada/matterloop/blob/main/...` 绝对链接，避免 PyPI 把相对地址解析
到错误位置。

## 翻译边界

翻译时必须保持以下内容逐字一致：

- Python 包名、导入名、类、函数、字段和枚举值；
- 配置默认值、数值限制、环境变量名和异常类型；
- 代码示例的调用方式、协议签名和依赖方向；
- 安全边界、失败语义、幂等、CAS、lease 和资源所有权约束。

说明文字、标题、示例中的自然语言目标可以本地化。不要把英文版缩成摘要，也不要把尚未实现的能力
翻译成已经支持。

英文页面中的仓库内链接应优先指向英文镜像；没有语言版本的源码、许可证、外部标准和 API 文档
继续使用原链接。

## 校验

文档契约测试会检查：

- 每份公开中文 Markdown 都存在英文镜像并提供双向语言入口；
- 中英文标题层级和代码块数量一致；
- 两种语言中的相对链接、包 README 的 GitHub 绝对链接和 Markdown 锚点都能解析；
- 英文正文优先链接英文镜像，且除语言切换示例外不残留中文；
- 中英文包 README 都覆盖真实公共字段和关键构造参数。

修改文档后运行：

```bash
uv run pytest tests/test_documentation_contracts.py
uv run ruff format --check .
uv run ruff check .
```

字段、默认值或公共签名变化时，中英文页面必须一起更新。契约测试用于发现遗漏，不能代替人工检查
术语、语气和安全表述。
