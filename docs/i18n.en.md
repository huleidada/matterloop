# Documentation internationalization

MatterLoop maintains its public Markdown in Simplified Chinese and English. Internationalization applies to
user-facing documentation only; it does not change Python identifiers, configuration fields, event values, or the
language used in source comments.

## Locale and file conventions

Simplified Chinese retains the original path, and English uses a colocated `.en.md` mirror:

| Chinese | English |
| --- | --- |
| `README.md` | `README.en.md` |
| `CHANGELOG.md` | `CHANGELOG.en.md` |
| `docs/architecture.md` | `docs/architecture.en.md` |
| `matterloop-core/README.md` | `matterloop-core/README.en.md` |

Keeping Chinese at the original path prevents existing GitHub links, PyPI `project.readme` metadata, and external
references from breaking. Every new public Markdown document must add its English mirror in the same commit.

## Language entry points

The first line of each Chinese page is:

```markdown
简体中文 | [English](the-English-file)
```

The first line of each English page is:

```markdown
[简体中文](the-Chinese-file) | English
```

Repository-root documents, `docs/`, and examples may use relative links. Distribution READMEs are rendered as PyPI
project descriptions, so their language links use absolute
`https://github.com/huleidada/matterloop/blob/main/...` URLs. This prevents PyPI from resolving relative paths against
the wrong location.

## Translation boundaries

The following content must remain exact in both languages:

- Python distribution names, import names, classes, functions, fields, and enum values;
- configuration defaults, numeric limits, environment-variable names, and exception types;
- code-example calling conventions, protocol signatures, and dependency directions;
- security boundaries, failure semantics, idempotency, CAS, lease, and resource-ownership constraints.

Explanatory prose, headings, and natural-language goals inside examples can be localized. Do not reduce the English
page to a summary, and do not translate a deferred capability as if it were implemented.

Repository-internal links in an English page should prefer an English mirror. Source code, licenses, external
standards, and API documentation without locale variants continue to use their original links.

## Validation

Documentation contract tests verify that:

- every public Chinese Markdown document has an English mirror and both pages expose a language switch;
- Chinese and English pages have the same heading levels and code-block counts;
- relative links, distribution README GitHub URLs, and Markdown anchors resolve in both languages;
- English prose links to English mirrors and contains no Chinese outside language-switch examples;
- both language versions of distribution READMEs cover the real public fields and key constructor parameters.

Run the following after changing documentation:

```bash
uv run pytest tests/test_documentation_contracts.py
uv run ruff format --check .
uv run ruff check .
```

When a field, default, or public signature changes, update both languages together. Contract tests detect omissions;
they do not replace a human review of terminology, tone, and security statements.
